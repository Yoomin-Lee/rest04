## 1. GitHub Actions 배포 환경변수 누락 수정

### 문제
`npm run deploy` (로컬 빌드 → gh-pages 푸시) 외에, GitHub Actions `deploy.yml`이 `main` 브랜치 push 시마다 자동 빌드·배포 중이었음.  
Actions 워크플로우는 `.env` 파일이 없는 CI 환경에서 빌드 → `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`가 `undefined`로 번들에 포함 → Supabase 클라이언트 초기화 실패 → 빈 화면.

### 원인 파악 방법
- 배포된 `index.html`에서 실제 로드되는 JS 파일명 확인
- 해당 JS 번들에서 Supabase 프로젝트 URL(`kbjxjogmnwurxbxnpfsz`) 검색 → 없음
- gh-pages 브랜치 JS(로컬 빌드본)에는 있었지만, Actions 빌드본이 Pages를 덮어쓰고 있었음

### 해결
1. GitHub Secrets 등록
   ```
   VITE_SUPABASE_URL=https://kbjxjogmnwurxbxnpfsz.supabase.co
   VITE_SUPABASE_ANON_KEY=eyJhbGci...
   ```
2. `.github/workflows/deploy.yml` 빌드 스텝에 `env:` 블록 추가
   ```yaml
   - run: npm run build
     env:
       VITE_SUPABASE_URL: ${{ secrets.VITE_SUPABASE_URL }}
       VITE_SUPABASE_ANON_KEY: ${{ secrets.VITE_SUPABASE_ANON_KEY }}
   ```

## 2. 냥젤리 커서 보라색 호버 적용

### 구현
- `public/cursor-purple.svg` 신규 생성 — 기존 커서의 `#111` fill을 `#7c3aed`(primary-600)로 변경
- `src/index.css`에 아래 규칙 추가
  ```css
  a, a *,
  button, button *,
  .tab-btn, .tab-btn *,
  [role="tab"], [role="tab"] *,
  label[for], label[for] * {
    cursor: url('/rest04/cursor-purple.svg') 11 17, pointer;
  }
  ```

### 트러블슈팅 — 커서 변경이 안 되는 문제
- 1차 시도: `a, button, .tab-btn`만 지정 → 동작 안 함
- 원인: `*` 셀렉터가 `a`, `button` 안의 **자식 요소**(`<span>`, `<svg>` 등)에도 검정 커서를 **직접** 적용. CSS에서 직접 지정된 값은 부모에서 상속된 값을 이김 → 자식 요소 위에서는 보라색 커서가 덮어씌워짐.
- 해결: `a *, button *, .tab-btn *` 등 자식 셀렉터까지 함께 지정

### 커서 크기 축소 및 하단 잘림 수정
- 기존 SVG: `width="28" height="28" viewBox="0 0 28 28"` — 메인 패드 하단이 `y=29`까지 내려가 viewBox 밖으로 잘림
- 수정: `width="22" height="23" viewBox="0 0 28 30"` — 높이를 30으로 늘려 잘림 방지, 렌더 크기를 22×23으로 축소
- CSS 핫스팟도 새 렌더 크기 기준으로 `14 21` → `11 17`로 조정

## 4. 라이트 모드 네비게이션 바 색상 개선

### 문제 구조
헤더는 두 가지 상태를 가짐
- **solid** (`scrolled || !isHome`): 스크롤 후 또는 서브 페이지
- **non-solid**: 홈 화면 최상단 (스크롤 전)

| 상태 | 기존 (라이트) | 문제 |
|------|-------------|------|
| non-solid | `bg-transparent` + 흰 텍스트 | 흰 body 배경에 흰 글자 → 안 보임 |
| non-solid (수정1) | `bg-deep-950/50` + 흰 텍스트 | `#080711` 50% 투명도가 흰 배경 위에 얹혀 회색으로 보임 |
| solid | `bg-white/95` + 회색 텍스트 | `backdrop-blur-md`가 콘텐츠를 블러해 회색처럼 보임 |

### 최종 해결
```
solid     → bg-white shadow-sm              (라이트)
           dark:bg-deep-900/95 dark:backdrop-blur-md  (다크)

non-solid → bg-white                        (라이트)
           dark:bg-deep-950/50 dark:backdrop-blur-sm  (다크)
```

텍스트 색상 (solid·non-solid 공통, 라이트 모드):
- 로고·아이콘: `text-primary-600` / `text-primary-700`
- 네비 링크 (활성): `text-primary-600`
- 네비 링크 (비활성): `text-primary-700 hover:text-primary-500`
- 게시판·유저명: `text-primary-700` / `text-primary-600`
- 로그아웃 버튼: `bg-primary-50 text-primary-700`
- 햄버거 아이콘: `bg-primary-700`

다크 모드 텍스트는 기존 흰색 계열 유지.

## 5. 드롭다운 메뉴 수평 정렬 수정

### 문제
네비게이션 항목 호버 시 드롭다운이 네비 텍스트와 수평이 맞지 않음.

### 원인
드롭다운 컨테이너가 `absolute left-1/2 -translate-x-1/2`로 `<li>` 중앙 기준 정렬.  
`<li>` 폭이 텍스트 + `px-5` 패딩 정도라 좁은데, `min-w-[10rem]` 드롭다운이 중앙에서 양쪽으로 펼쳐지면서 왼쪽으로 튀어나옴 → 네비 텍스트 시작점(`px-5`)과 드롭다운 텍스트 시작점이 불일치.

### 해결
`left-1/2 -translate-x-1/2` → `left-0` 으로 변경.  
`<li>` 좌측 끝 기준으로 정렬하면, 네비 링크(`px-5`)와 드롭다운 항목(`px-5`) 모두 같은 x 위치에서 텍스트가 시작됨.

```jsx
// 변경 전
<div className="absolute left-1/2 top-full z-50 -translate-x-1/2 pt-1">

// 변경 후
<div className="absolute left-0 top-full z-50 pt-1">
```

`src/components/Header.jsx`
