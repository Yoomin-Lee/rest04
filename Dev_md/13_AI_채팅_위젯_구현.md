# 13. AI 채팅 위젯 구현

작업일: 2026-06-17

---

## 목표

메인 페이지 오른쪽 하단에 AI 채팅 팝업 창 추가.  
Solar API(Upstage)와 OpenAI API를 사용하여 반려동물 교육 Q&A 응답 제공.  
API 키는 Supabase Edge Function Secrets에 저장하여 클라이언트에 노출되지 않도록 처리.

---

## 구현 내용

### 채팅 위젯 (`src/components/ChatWidget.jsx`)

- 오른쪽 하단 고정 버튼 (`fixed bottom-24 right-8 z-50`)
- 클릭 시 480×500px 팝업 창 열림
- **Solar / OpenAI 탭 토글** — 사용자가 직접 모델 선택
- 응답 메시지에 어떤 모델이 답했는지 라벨 표시
- 로딩 중 점 3개 애니메이션 (bounce)
- Enter 전송, Shift+Enter 줄바꿈
- 다크모드 대응

### Supabase Edge Function (`supabase/functions/chat/index.ts`)

- 요청 body: `{ messages, model }` (`model`: `'solar'` | `'openai'`)
- `model !== 'openai'`이면 Solar API 우선 호출
- Solar 실패 또는 `model === 'openai'`이면 OpenAI GPT-4o-mini 사용
- 응답에 `model` 필드 포함하여 어떤 API가 응답했는지 반환
- CORS 처리 포함 (`Access-Control-Allow-Origin: *`)
- API 키: Supabase Secrets에 `SOLAR_API_KEY`, `OPENAI_API_KEY` 저장

### App.jsx

`<ChatWidget />` 컴포넌트 추가

## 트러블슈팅

### Edge Function 404 / ERR_FAILED 문제

Supabase 대시보드 Code 탭에서 코드를 저장해도 실제 Edge 네트워크에 배포되지 않는 문제.  
→ Supabase CLI로 직접 배포하여 해결:

```bash
npx supabase@latest login --token <ACCESS_TOKEN>
npx supabase@latest functions deploy chat --no-verify-jwt --project-ref kbjxjogmnwurxbxnpfsz
```

### GitHub Push Protection 차단

`.env`에 API 키를 직접 넣어 빌드하면 JS 번들에 포함되어 GitHub가 push 차단.  
→ Supabase Edge Function Secrets 방식으로 전환하여 해결.

### JWT Verification

Edge Function 호출 시 CORS preflight OPTIONS 요청이 401로 차단되는 문제.  
→ `--no-verify-jwt` 플래그로 배포하여 해결.
